# 文件管理系統 ASP.NET WebForms 後端（改良版 v2）

本目錄包含文件管理系統 ASP.NET WebForms 後端的改良版原始碼，對應桌面版本的全面安全性、資料一致性與功能補強。

---

## 改良摘要

### A 類：安全性修補

| 檔案 | 改良內容 |
|------|----------|
| `審核管理.aspx.cs` | **A1** 審核身分改由帳號角色（Session）判斷，不再依賴可偽造的 IP 位址；管理員可審核所有組別，負責人只能審核自己組別 |
| `審核管理.aspx.cs` | **A2** `IN (...)` 子句改為只含整數 ID 的安全拼接，消除 SQL Injection 風險 |
| `檔案上傳.aspx.cs` | **A3** 統一呼叫 `檔案安全輔助.驗證上傳檔案()`，涵蓋雙重副檔名偽裝、MIME 比對、危險副檔名等完整檢查，移除舊版重複且較簡陋的驗證邏輯 |
| `App_Code/帳號輔助.cs` | **A4** 新增 `角色輔助.要求負責人以上()` 方法，統一頁面權限保護入口 |

### B 類：資料一致性修補

| 檔案 | 改良內容 |
|------|----------|
| `App_Code/資料夾輔助.cs` | **B1** 新增 `重新命名資料夾()` 方法：先移動實體磁碟目錄，成功後才更新 DB，並遞迴同步所有子孫資料夾的 `實體路徑` 與子孫檔案的 `檔案路徑`；磁碟操作失敗時記錄資安警示並回傳 `false`，不更新 DB |
| `檔案瀏覽.aspx.cs` | **B1** 改名動作改呼叫新的 `重新命名資料夾()` 方法，並補寫操作紀錄 |
| `審核管理.aspx.cs` | **B2** 審核未通過時，實體移檔失敗不再更新 DB 路徑（避免 DB 與磁碟不一致）；改為記錄資安警示，DB 路徑維持原值 |
| `資源回收桶.aspx.cs` | **B3** 還原動作補實體移檔：將檔案從回收桶路徑移回時效區對應組別路徑，並更新 DB 的 `檔案路徑`；移檔失敗時記錄警示並取消還原，不留殭屍紀錄 |

### C 類：功能補強

| 檔案 | 改良內容 |
|------|----------|
| `App_Code/輔助類別.cs` | **C1** `操作紀錄輔助.記錄()` 已完整支援寫入 `帳號編號` 與 `登入帳號` 欄位，所有操作均可追蹤到具體帳號 |
| `首頁.aspx.cs` | **C2** 統計數量（永久區/時效區/待審核/即將到期）、今日下載、最近上傳、最近操作依角色範圍過濾：非管理員只看自己組別數據 |
| `首頁.aspx.cs` | **C3** 最近操作紀錄補顯示操作者姓名（JOIN 使用者帳號），不再只顯示 IP |
| `檔案上傳.aspx.cs` | **C4** 上傳成功訊息從 `AppSettings["時效區天數"]` 讀取天數，不再寫死「30天」 |
| `資源回收桶.aspx.cs` | **C5** 還原天數從 `AppSettings["時效區天數"]` 讀取，不再寫死 |

## 資料庫升級

執行 `資料庫升級_v2.sql` 以補充必要欄位與索引：

```sql
-- 補充 操作紀錄 的帳號欄位
ALTER TABLE [操作紀錄] ADD [帳號編號] INT NULL, [登入帳號] NVARCHAR(50) NULL;

-- 補充 資料夾 的實體路徑與刪除欄位
ALTER TABLE [資料夾] ADD [實體路徑] NVARCHAR(1000) NULL, ...;

-- 補充 檔案主檔.資料夾編號 欄位（若尚未存在）
ALTER TABLE [檔案主檔] ADD [資料夾編號] INT NULL;
```

> **注意**：執行前請先備份資料庫。腳本採用 `IF NOT EXISTS` 保護，可安全重複執行。
